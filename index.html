The Problem

AI agents waste 90% of tokens talking to each other in English

When two LLMs communicate, they don't need grammar, politeness, or redundancy. They need meaning. Ogenti teaches them to strip everything else away.

💸

Token Waste

Multi-agent systems burn through API tokens with natural language overhead. Every "please", "sure", and formatting token is money on fire.

🐌

Latency

More tokens = more time. Agent-to-agent conversations that could take 3 tokens take 30. Multiply across thousands of calls per day.

🔒

No Standard Protocol

There's no TCP/IP for AI communication. Every multi-agent framework invents its own format. None of them learn from usage.

Architecture

5-phase progressive training

Agents evolve through increasingly complex communication challenges. Each phase builds on the last — like learning to speak, then write, then compress.

0
Phase 0 — 5K episodes

Warmup

Supervised pre-training. Encoder learns to map natural language to initial protocol tokens. Decoder learns to reconstruct meaning.

Target:  2x compression
1
Phase 1 — 15K episodes

Simple Communication

Reinforcement learning kicks in. Two agents start negotiating their own token meanings through reward-driven optimization.

Target:  5x compression, 70% fidelity
2
Phase 2 — 20K episodes

Complex Multi-Agent

Three agents juggle multiple tasks simultaneously. The protocol must handle ambiguity, context-switching, and multi-hop reasoning.

Target:  8x compression, 85% fidelity
3
Phase 3 — 10K episodes

Generalization

All 12 task categories activated. The protocol learns to handle anything — code, math, creative writing, translation — with the same token vocabulary.

Target:  10x compression, 90% fidelity
4
Phase 4 — 8K episodes

Universal Adapter

Knowledge distillation extracts the learned protocol into a lightweight ~3MB adapter. Any LLM can now speak the protocol without fine-tuning.

Target:  ~3MB portable adapter
